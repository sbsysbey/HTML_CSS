Javada MAPS konusu

Java inheritence sınıfındadırlar.
  • Key ve value olarak ikili yapıdadırlar.
    • {Türkiye=Ankara} gibi
  • {} Curly Braces yapısı kullanılır.

    • {null=Aysel Ay, 101=Kamil Kam, 102=Ali Yazar,108=Veli Bozar} gibi
  • Key'ler tekrarli deger kabul etmez. (Unique olmalidirlar/Kimlik numaralari gibi).
  • Value'lar tekrarli olabilirler. (Isimler)
    • {null=Aysel Ay, 102=Veli Can,103=Veli Can} gibi

HASHMAP

  • En cok kullanilani, en hizlilaridir. iş arttıkça hız yavaşlar.dolayısıyla az özelliği vardır.
  • Hizli olabilmek icin siralamayi rastgele yapar.
  • Hizli olabilmek icin synchronize olmazlar ve thread-safe degildirler.
    • (telefon misali. aynı anda farklı şeyleri yapabilmesi.)
  • Bir tane key null olabilir, birden fazla value null olabilir.
  • Oluşturulması:
    • HashMap '<'String, String'>' isim = new HashMap();
TREEMAP
  • TreeMap elemanlari natural order'a gore siralar.
  • TreeMap, Map'lerin en yavaşıdır.

  • TreeMap'lerde key'ler icin "null" kullanılmaz.
  • TreeMap'lerde value'lar icin "null" istenildigi kadar kullanilabilir.
  • TreeMap'ler syncronized ve thread-safe degildirler.
  • FirstKey ve Last key komutları kullanılabilir.
  • Oluşturulması:

    • TreeMap '<'String, String'>'' isim = new TreeMap '<'String, String'>'();
LİNKEDHASHMAP
  • LinkedHashMap elemanlari Giriş sırasına göre depolar.
  • Bir null key ve ve birden çok null value olabilir.

  • Ekleme sırasını koruyan ek bir özelliğe sahip HashMap gibidir.

  • Oluşturulması;

    • {LinkedHashMap '<'String, String'>'' isim = new LinkedHashMap'<'String, String'>''();{}
    HASHTABLE
    • HashMap ve TreeMap synchronized ve thread-safe olmadıklarından Map'ler arasindan synchronized ve thread-safe olan bir map'e ihtiyacı karşılamak üzere oluşturulmuştur.
      Yani synchronized ve thread-safe dirler.
    • HashTable ne key icin ne de value icin "null" kullanmaz.
    • HashTable elemanlari rastgele siralar.

    Oluşturulması:

    HashTable'<'String, String'>'' isim = new Hashtable'<'String, String'>'();
